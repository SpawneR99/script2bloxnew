Script Troubleshooting Guide — Common Issues & Solutions

Script Not Loading After Execution

Pressed Execute and got nothing? Give it 10-15 seconds. Arise scripts with dungeon and raid modules take time to initialize their Lua bytecode interpreter and render the full GUI. Still blank? Confirm injection succeeded, re-inject Delta, check you're on the latest build, and verify the script supports the current game version.

GUI Not Appearing or Appearing Off-Screen

GUIs can spawn off-screen on non-standard resolutions. Most scripts bind a toggle key, usually Home or Right Shift on PC, or a floating button on mobile. Try those keys before assuming the script failed.

Features Not Working Despite GUI Loading

If the GUI renders but features don't respond, the script likely needs patching for the latest Arise update. Check RoCrack for a newer version. Also confirm the game was fully loaded before you injected... executing during a loading screen causes partial feature failures more often than most people realize.

Game Crashing After Script Execution

Crashes are usually caused by memory hooking conflicts or too many features enabled simultaneously on hardware that can't handle the load. Disable heavy features, re-enable one at a time, and verify your device has enough RAM for the script.

Script Working Inconsistently

Inconsistent behavior is almost always a timing issue. Inject and execute after the game is fully loaded, not during transitions. Server variation can also cause problems, so hop to a different server if features work on one but fail on another.
