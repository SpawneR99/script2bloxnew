Script Troubleshooting Guide — Common Issues & Solutions

Script Not Loading After Execution

If nothing happens after you hit Execute in Delta, give it 10-15 seconds. Some scripts need time to initialize their Lua bytecode interpreter and render the GUI. Still nothing? Confirm injection was successful, try re-injecting, check you're on the latest Delta build, and verify the script matches the current game version.

GUI Not Appearing or Appearing Off-Screen

GUIs sometimes spawn outside visible screen bounds, especially at non-standard resolutions. Most scripts bind a key to toggle or reset the GUI position, usually Home or Right Shift on PC, or a floating button on mobile. Try pressing those before assuming the script failed.

Features Not Working Despite GUI Loading

When the GUI shows up but individual features don't respond, the script likely needs a patch for the latest game update. Check RoCrack for a newer version. Also confirm you waited for the game to fully load before injecting... executing during a loading screen is a common cause of partial failures.

Game Crashing After Script Execution

Crashes usually mean a script is conflicting with specific game mechanics, or too many features are running simultaneously on a device that can't handle the memory overhead. Disable heavy features, enable them one at a time to isolate the problem, and check that your hardware meets the script's minimum specs.

Script Working Inconsistently

Inconsistent behavior almost always traces back to timing. Inject and execute after the game is fully loaded, not during transitions or loading screens. Server-side variations can also cause this, so try hopping to a different server if features work on one but not another.
