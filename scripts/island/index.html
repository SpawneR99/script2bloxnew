Script Troubleshooting Guide — Common Issues & Solutions

Script Not Loading After Execution

Hit Execute and nothing happened? Give it 10-15 seconds first, some scripts need time for the Lua bytecode interpreter to initialize. If still nothing, check that injection succeeded before you executed. Try re-injecting Delta and running the script again. Make sure you are on the latest Delta version and that the script supports the current game patch.

GUI Not Appearing or Appearing Off-Screen

GUIs sometimes spawn outside your visible screen area, especially on non-standard display resolutions. Most scripts bind a toggle key, usually Home or right Shift on PC, or a floating button on mobile. Press those to bring the GUI back into view or reset its position.

Features Not Working Despite GUI Loading

GUI loads fine but toggles do nothing? The script probably needs a patch for the latest game version. Check RoCrack for an updated build. Also confirm you waited for the game to fully load before injecting... executing during loading screens causes this exact issue.

Game Crashing After Script Execution

Crashes usually mean the script is conflicting with a specific game mechanic or you have too many features enabled at once on a device that cannot handle it. Disable heavy features, enable them one at a time to isolate the culprit, and make sure your device meets the minimum specs for the script you are running.

Script Working Inconsistently

Works sometimes, fails other times? Timing is almost always the problem. Inject and execute only after the game world is completely loaded, never during transitions or loading screens. Server-side differences can cause inconsistency too. Try hopping to a different server.
